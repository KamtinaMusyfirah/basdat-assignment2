01

--------

Evaluation Plan

PostgreSQL, Oracle, SQL Server, MySQL

Sebagian besar sistem database menyediakan cara untuk melihat evaluation plan yang dipilih untuk dijalankan query tertentu. Biasanya yang terbaik adalah menggunakan GUI yang disediakan dengan sistem database untuk melihat evaluation plan. Namun, jika menggunakan antarmuka baris perintah, banyak database mendukung variasi dari perintah explain (query), yang menampilkan execution plan yang dipilih untuk query yang ditentukan . Sintaks yang bervariasi dengan database yang berbeda:

  1. PostgreSQL

    PostgreSQL menggunakan syntax explain (query) sama seperti dengan MySQL

  2. Oracle

    Oracle menggunakan syntax explain plan for, namun syntax tersebut hanya menyimpan file resultant plan dalam tabel yang disebut plan_table. Query "select * from table(dbms xplan.display);" digunakan untuk menampilkan plan yang disimpan.

  3. SQL Server

    SQL Server membutuhkan syntax set showplan text on untuk dieksekusi sebelum mengirimkan query. Kemudian, saat query dikirim , bukannya menjalankan query, tetapi evaluation plan ditampilkan.

  4. MySQL

    MySQL menggunakan syntax explain (query) seperti PostgreSQL, tetapi outputnya merupakan sebuah tabel yang isinya tidak mudah dimengerti. Namun, menjalankan show warning setelah command expalin menampilkan Evaluation Plan dalam format yang lebih mudah dibaca manusia.

Evaluation Plan

Contoh

Penjelasan Tabel

menunjukkan bahwa pada nomor urut berapa SELECT tersebut dalam query . dan pada tabel diatas menujukkan bahwa kedua tabel berada pada nomor urut 1.

id

pada tabel diatas yaitu PRIMARY artinya bahwa tabel tersebut berada diluar select sedangkan MATERIALIZED artinya bahwa subquery terwujud.

select_type

merujuk pada nama table database tersebut yaitu table mahasiswa dan table penerima_beasiswa.

table

pada kolom ini dijelaskan bagaimana kolom tersebut bergabung . ALL berarti bahwa pemindaian tabel lengkap sedangkan nilai index fungsinya hampir sama dengan ALL tetapi menggunakan index dalam pemindaiannya.

type

menunjukkan index dari mana mysql memilih untuk menemukan table dari table tersebut . jika nilai pada kolom tersebut NULL maka index nya tidak relevan sedangkan jika primary

possible_keys

menunjukkan index yang diputuskan untuk digunakan pada mysql tersebut . jika nilai pada kolom tersebut NULL maka hal tersebut berarti bahwa mysql tidak dapat menemukan index yang digunakan untuk mengeksekusi query secara lebih efisien

key

menunjukkan panjang kunci yang diputuskan untuk digunakan. Nilai dari key_len memungkinkan untuk menentukan berapa banyak bagian yang digunakan oleh mysql . jika nilai pada kolom key adalah NULL maka pada kolom key_len juga akan NULL.

key_len

menujukkan kolom atau konstanta yang dibandingkan dengan index yang disebutkan dalam kolom ley untuk memilih baris dari table .

ref

menunjukkan jumlah baris mysql yang harus diperiksa untuk mengeksekusi query. Jika menggunakan InnoDB , maka angka pada kolom tersebut adalah perkiraan dan tidak selalu tepat.

rows

menunjukkan informasi tambahan tentang bagaimana mysql menyelesaikan query tersebut . nilai pada tabel tersebut adalah using index berarti bahwa informasi kolom diambil dari tabel hanya menggunakan informasi dari index tanpa membaca baris yang ada. Using where berarti bahwa digunakannya kondisi WHERE .
